# Licensing outreach templates

Send both in parallel — the answers from each inform your provider choice in `providers.py`.

## To PSX

**To:** marketdatarequest@psx.com.pk
**Subject:** Request for authorization / clarification for delayed PSX quote display on public website

```
Dear PSX Market Data Team,

I am the owner of a public website project that tracks Pakistan Stock Exchange
securities and currently publishes end-of-day market snapshots for informational
purposes.

We are planning a new intraday feature that would display approximately real-time
or delayed quotes for a limited set of PSX-listed symbols on our public website.
The feature would show clear timestamps, indicate that data may be delayed, and
would not support trading, order entry, or redistribution to third parties.

Could you please confirm:
1) Whether public website display of delayed PSX quotes requires a data license
   or written authorization from PSX.
2) Whether displaying limited delayed quote fields (symbol, last price, change,
   volume, timestamp) is permitted without a commercial market-data agreement.
3) What attribution, disclaimer, and delay wording PSX requires for this use case.
4) If licensing is required, which product or agreement applies and what the
   next steps are.

I would be happy to provide screenshots or a short product description.

Kind regards,
[Your Name]
[Website / Repository]
[Email]
```

## To Capital Stake

**To:** data.solutions@capitalstake.com
**Subject:** Inquiry about licensed delayed / real-time PSX API for public website display

```
Dear Capital Stake Data Solutions team,

I maintain a public PSX-tracking website and am evaluating a compliant intraday
quotes integration.

I would like to understand the commercial and licensing options for:
1) delayed public website quote display,
2) real-time public website quote display,
3) REST versus WebSocket access,
4) allowable redistribution and attribution terms,
5) entry-level pricing for a small public informational website.

My preferred technical shape is a backend API that serves a limited watchlist of
PSX symbols and pushes updates to a browser frontend.

If possible, please share:
- pricing tiers,
- minimum contract requirements,
- rate limits / connection limits,
- redisplay rights,
- any required PSX approvals,
- sandbox / trial availability.

Thank you,
[Your Name]
[Website / Repository]
[Email]
```
